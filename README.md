# 🛍️ DeveloperShop – SQL Database Design & Business Analysis Project

A beginner-friendly SQL project that simulates the backend database of an e-commerce merchandise store for developers. The project demonstrates relational database design, table relationships, SQL querying, and basic business analytics using MySQL.

Although the dataset is intentionally small for learning purposes, the project follows industry-standard database design principles and provides a strong foundation for more advanced SQL analytics.

---

## 📖 Project Overview

DeveloperShop is an online merchandise store selling products for developers such as hoodies, mugs, notebooks, stickers, T-shirts, and accessories.

The project focuses on:

- Relational Database Design
- Database Normalization
- Primary & Foreign Keys
- One-to-Many Relationships
- SQL Data Definition Language (DDL)
- SQL Data Manipulation Language (DML)
- Business Intelligence Queries
- Revenue & Sales Analysis

# 🗄️ Database Schema

The database consists of five interconnected tables.

## 1. Customers

Stores customer information.

| Column | Description |
|---------|-------------|
| customer_id | Primary Key |
| name | Customer Name |
| email | Unique Email Address |
| city | Customer City |
| signup_date | Registration Date |

---

## 2. Products

Stores all available products.

| Column | Description |
|---------|-------------|
| product_id | Primary Key |
| product_name | Product Name |
| category | Product Category |
| price | Selling Price |
| stock | Inventory Available |

---

## 3. Orders

Stores customer orders.

| Column | Description |
|---------|-------------|
| order_id | Primary Key |
| customer_id | Foreign Key |
| order_date | Date of Purchase |
| order_status | Delivered / Pending / Cancelled |

---

## 4. Order Items

Stores products purchased in every order.

| Column | Description |
|---------|-------------|
| order_item_id | Primary Key |
| order_id | Foreign Key |
| product_id | Foreign Key |
| quantity | Quantity Purchased |

---

## 5. Payments

Stores payment details.

| Column | Description |
|---------|-------------|
| payment_id | Primary Key |
| order_id | Foreign Key |
| payment_mode | Payment Method |
| amount | Amount Paid |
| payment_date | Payment Date |

# 📈 Business Analysis Performed

## 1️⃣ Total Revenue

Calculates the overall revenue generated from successful payments.

```sql
SELECT SUM(amount) AS total_revenue
FROM payments;
```

---

## 2️⃣ Revenue by Product

Calculates revenue generated by each product.

```sql
SELECT
    p.product_name,
    SUM(oi.quantity * p.price) AS revenue
FROM order_items oi
JOIN products p
ON oi.product_id = p.product_id
JOIN orders o
ON oi.order_id = o.order_id
WHERE o.order_status='Delivered'
GROUP BY p.product_name
ORDER BY revenue DESC;
```

---

## 3️⃣ Top Customers by Spending

Finds customers who have spent the highest amount.

```sql
SELECT
    c.name,
    SUM(p.amount) AS total_spent
FROM customers c
JOIN orders o
ON c.customer_id=o.customer_id
JOIN payments p
ON o.order_id=p.order_id
GROUP BY c.name
ORDER BY total_spent DESC;
```

---

## 4️⃣ Best Selling Products

Calculates product demand.

```sql
SELECT
    p.product_name,
    SUM(oi.quantity) AS total_sold
FROM order_items oi
JOIN products p
ON oi.product_id=p.product_id
GROUP BY p.product_name
ORDER BY total_sold DESC;
```

---

## 5️⃣ Cancelled Orders

Counts cancelled orders.

```sql
SELECT COUNT(*) AS cancelled_orders
FROM orders
WHERE order_status='Cancelled';
```
